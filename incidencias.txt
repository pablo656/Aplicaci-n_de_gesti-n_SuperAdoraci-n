[C-01] mvc/model/model_user.php - Línea ~31
Tipo: Bug lógico - execute() doble
En iniciousuario(), el statement se ejecuta en la línea 19 con
comprobación de error, pero vuelve a llamarse execute() de forma
incondicional en la línea 31 sin comprobar el resultado. Puede causar
comportamiento inesperado o errores silenciosos.

[C-02] mvc/model/model_reservas.php - Línea ~145
Tipo: Error en bind_param - parámetro de más
En eliminar_reserva_por_id_usuario(), bind_param usa "ii" (dos enteros)
pero solo se pasa un parámetro. Causará un fatal error al ejecutar.

[C-03] mvc/model/model_reservas.php - Líneas ~106, ~114, ~188
Tipo: Tipo incorrecto en bind_param
- Línea 106: usa "dii" pero $cantidad no es decimal, debería ser "iii"
- Línea 114: usa "di" pero $cantidad no es decimal, debería ser "ii"
- Línea 188: usa "ss" pero $id y $cantidad son enteros, debería ser "ii"
Puede causar conversiones silenciosas o errores con ciertos valores.

[C-04] mvc/vista/Sign_in.php - Líneas ~17-23
Tipo: Conflicto de merge sin resolver
El archivo contiene marcadores de conflicto de Git (<<<<<<, =======,
>>>>>>>). PHP no puede parsear el archivo y romperá el flujo de registro.

ALTAS (4)
===============================================================

[A-01] mvc/vista/administrador/productos-administrador.php - Línea ~43
Tipo: Falta echo en output PHP
El campo oculto tiene value="<?php $producto['id']?>" sin echo.
El id del producto nunca se envía en el formulario, rompiendo edición/borrado.
Debería ser: value="<?= $producto['id'] ?>"

[A-02] mvc/vista/administrador/IndexProducto-administrador.php - Línea ~4
Tipo: Error lógico en control de acceso
La comprobación de rol usa && (Y): deniega acceso solo si el usuario es
administrador Y dueño a la vez (imposible). Debe usar || (O) para denegar
cuando no es ninguno de los dos roles.

[A-03] mvc/controller/productoController.php - Línea ~94
Tipo: Parámetro faltante en llamada a método
En update_producto(), la llamada al modelo omite el parámetro $subcategoria
que sí existe en la firma del método del modelo. Puede causar actualización
incorrecta o error en la BD.

[A-04] mvc/model/model_productos.php / controller - Línea ~82
Tipo: Desajuste de parámetros en add_productos()
El orden de parámetros entre la llamada del controlador y la definición
del método del modelo no coincide. Puede causar que $categoria, $imagen
o $porcentaje_descuento se guarden en columnas incorrectas de la BD.

MEDIAS (6)
===============================================================

[M-01] mvc/helpers/Mailer.php - Líneas ~14-15
Tipo: Constantes no definidas
MAIL_USER y MAIL_PASS se usan en el constructor de Mailer pero están
comentadas en mail_config.php. El envío de correos fallará con un
"Undefined constant" notice o error.

[M-02] mvc/model/model_productos.php - Línea ~51
Tipo: Falta comprobación de null antes de acceso a array
En comprobar_stock(), se accede a $producto["stock"] sin verificar si
$producto es null. Si no hay resultado en la BD causará un error tipo
"Trying to access array offset on null".

[M-03] mvc/vista/IndexCarrito.php - Línea ~80
Tipo: Falta isset() antes de acceder a $_SESSION["id"]
Se accede a $_SESSION["id"] sin comprobar si está definido. Si el usuario
no ha iniciado sesión generará un PHP notice/warning.

[M-04] mvc/vista/catalogo.php - Línea ~36
Tipo: Inconsistencia en nombre de cookie
La vista catalogo.php referencia la cookie "reservas" pero IndexCarrito.php
gestiona también la cookie "pedidos". Puede haber inconsistencias en el
manejo del carrito para usuarios no registrados.

[M-05] mvc/vista/reservas.php - Línea ~6
Tipo: Typo en ruta de CSS
La ruta del stylesheet contiene "git_develover" en lugar de "git_developer".
El CSS del header no se cargará en la página de reservas.

[M-06] mvc/vista/layerHeader.php - Línea ~35
Tipo: HTML malformado
El enlace de Log out abre etiqueta <a> pero cierra con </i> en lugar de
</a>. HTML inválido que puede causar problemas de layout.

BAJAS (2)

[B-01] mvc/vista/IndexPerfil.php - Línea ~20
Tipo: Código muerto
count($pedidos) se llama sin asignar ni usar el resultado. Es código de
depuración que no tiene efecto.

[B-02] mvc/vista/IndexHome.php - Varias líneas
Tipo: Formato no estándar en header()
Algunos header("Location:...") omiten el espacio después de "Location:".
Técnicamente funciona pero no cumple el estándar HTTP (RFC 7230).
Ejemplo: "Location:indexHome.php" debería ser "Location: indexHome.php".
